Git

Clone into a new repository (without history)

Sometimes you want to reuse a repository as a starting point, but keep a completely clean history.

git clone https://github.com/original/project.git
cd project
rm -rf .git
git init
git remote add origin https://github.com/you/new-repo.git
git add .
git commit -m "Initial commit"
git push -u origin main
Removing .git also removes the original remote.
You must explicitly configure a new one before pushing.
First commit and push to a new repository

After creating a new repository (for example on GitHub), the first commit establishes the initial project state and connects it to the remote.

git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/you/project.git
git push -u origin main
The -u flag sets the upstream branch, allowing future git push
and git pull commands without specifying the remote.
Separate feature and documentation commits

Keeping functional changes and documentation updates in separate commits
improves readability and makes code reviews easier.

git restore --staged README.md
git add build/build-docs.ps1
git commit -m "feat(build): add multi-workbook support"
This approach makes the intent of each commit explicit
and results in a cleaner project history.
Mulitple feature and documentation commits

Keeping multiple functional changes and documentation update in separate commits
improves readability and makes code reviews easier.

git add .
git restore --staged README.md

Check status of staged files

git status
git diff --staged
git commit -m "feat(build): add multi-workbook support"
git add README.md
git commit -m "docs: update README for v0.6.0"
This approach makes the intent of each commit explicit
and results in a cleaner project history.
Create and push a release tag

Tags are commonly used to mark release points in the repository history.
They can also be created after the fact.

git tag v0.6.0
git push --tags
Tags are lightweight references and do not modify the commit history.
Creating them later is perfectly fine.
